# Клітинний автомат для моделювання росту ракових пухлин
## Пояснення моделі, формул та обчислень

---

## 1. Що таке клітинний автомат?

Клітинний автомат (Cellular Automaton, CA) — математична модель, яка складається з **сітки клітин**, де кожна клітина перебуває в одному зі станів, і стан кожної клітини змінюється на кожному кроці за **набором правил**, що залежать від стану її сусідів.

Формальний запис: CA = **(C, n, S, f)**

| Символ | Назва | Що це |
|--------|-------|-------|
| **C** | Множина клітин | 2D решітка N×N: C = {(i,j) : 0 ≤ i,j < N} |
| **n** | Функція сусідства | Сусідство Мура — 8 сусідів навколо кожної клітини |
| **S** | Множина станів | {EMPTY, HEALTHY, CANCER, CANCER_STEM, IMMUNE, NECROTIC} |
| **f** | Функція переходу | Стохастичні (ймовірнісні) правила зміни стану |

---

## 2. Решітка та стани клітин

Решітка — двовимірний масив `numpy` розміром N×N (за замовчуванням 100×100), де кожна клітина зберігає ціле число від 0 до 5:

```
· = EMPTY        (0) — порожнє місце
○ = HEALTHY      (1) — здорова клітина тканини
● = CANCER       (2) — звичайна ракова клітина (RTC)
★ = CANCER_STEM  (3) — стовбурова ракова клітина (STC)
◆ = IMMUNE       (4) — імунна клітина
✕ = NECROTIC     (5) — некротична (мертва) клітина
```

Разом з решіткою існують два допоміжні масиви:
- `potential[r][c]` — скільки поділів залишилось у клітині (для RTC: 0..pmax, для STC: pmax+1 як маркер безсмертності)
- `necrotic_timer[r][c]` — лічильник часу розпаду некротичної клітини

---

## 3. Сусідство Мура

Кожна клітина (r, c) має до 8 сусідів — по горизонталі, вертикалі та діагоналях:

```
(r-1,c-1)  (r-1,c)  (r-1,c+1)
 (r, c-1)   (r,c)    (r, c+1)
(r+1,c-1)  (r+1,c)  (r+1,c+1)
```

На краях решітки кількість сусідів менша (у кутку — лише 3). Реалізовано через `get_neighbors()`, яка фільтрує координати за межами сітки.

---

## 4. Стани клітин

### EMPTY (0)
Порожнє місце. Сюди може поділитись або переміститись інша клітина.

### HEALTHY (1)
Здорова клітина тканини. Пасивна — нічого не робить, але може бути витіснена раковою клітиною при поділі або міграції.

### CANCER — RTC (2)
**Regular Tumor Cell** — звичайна ракова клітина.

Має **обмежений проліферативний потенціал** `p = pmax = 10`. Після кожного поділу `p` зменшується на 1 у обох клітин (батьківської і дочірньої). Коли `p = 0` — клітина гине (→ NECROTIC).

```
RTC(p=3) ділиться → RTC(p=2) + дочірня RTC(p=2)
RTC(p=1) ділиться → RTC(p=0) + дочірня RTC(p=0)
RTC(p=0) → наступного кроку гине → NECROTIC
```

### CANCER_STEM — STC (3)
**Stem Tumor Cell** — стовбурова ракова клітина.

- **Безсмертна**: не гине від апоптозу, потенціал не вичерпується (зберігається як `pmax + 1`)
- При поділі:
  - з ймовірністю **Ps** → дочірня STC (симетричний поділ)
  - з ймовірністю **1 − Ps** → дочірня RTC з `p = pmax` (асиметричний поділ)

```
STC ділиться:
  ├─ Ps (1%)   → нова STC (безсмертна)
  └─ 1-Ps (99%) → нова RTC (p = pmax, смертна)
```

### IMMUNE (4)
Імунна клітина. Активно шукає ракові клітини серед сусідів і атакує їх. Якщо ракових немає — мігрує. Нові імунні клітини рекрутуються на краях решітки.

### NECROTIC (5)
Мертва клітина. Залишається на місці `necrotic_decay_time = 50` кроків, потім стає EMPTY.

---

## 5. Функція переходу станів (функція f)

### 5.1. Ракова клітина (RTC або STC)

За один крок — **одна** дія з чотирьох (перевіряються послідовно):

```
1. Апоптоз?     rand < PA  → гине (→ NECROTIC)       [тільки для RTC]
2. Потенціал?   p ≤ 0      → гине (→ NECROTIC)       [тільки для RTC]
3. Поділ?       rand < PP  → ділиться на вільне місце
4. Міграція?    rand < Pm  → переміщується на вільне місце
5. Спокій       → нічого
```

Кроки 3 і 4 виконуються лише якщо є хоча б одне вільне (EMPTY або HEALTHY) місце серед сусідів. Порядок сусідів перемішується випадково перед кожною дією.

### 5.2. Імунна клітина

```
Є ракові сусіди?
  ТАК → бій:
    rand < pdT           → рак гине (→ NECROTIC)
    pdT ≤ rand < pdT+pdI → IC гине (→ EMPTY)
    інакше               → нічого
  НІ → міграція:
    rand < immune_migration_speed → переміщується на EMPTY місце
```

### 5.3. Некротична клітина

```
necrotic_timer += 1
Якщо timer ≥ necrotic_decay_time (50) → стає EMPTY
```

### 5.4. Здорова клітина

Нічого не робить (пасивна).

## 6. Параметри та формули

### Ймовірність проліферації (PP)

```
PP = Δt / CCT = 1 день / 24 год ≈ 0.0417 (≈4%)
```

Кожна ракова клітина має ~4% шанс поділитись за один крок (= 1 день). Відповідає реальному часу клітинного циклу ~24 години.

### Ймовірність апоптозу (PA)

```
PA = 0.005 (0.5%) для RTC
PA = 0             для STC  ← стовбурові безсмертні
```

### Ймовірність міграції (Pm)

```
Pm = 0.02 (2%)
```

### Ймовірність симетричного поділу STC (Ps)

```
Ps = prob_stem_division = 0.01 (1%)
```

Навіть 1% достатньо щоб кількість STC поступово зростала, роблячи пухлину дедалі агресивнішою.

### Проліферативний потенціал

```
Нова RTC:         p = pmax = 10
Після поділу:     p_батько = p_дочірня = p_старе - 1
Якщо p = 0:       клітина гине наступного кроку
```

### Таблиця всіх параметрів

| Параметр | Значення за замовч. | Опис |
|----------|---------------------|------|
| `grid_size` | 100 | Розмір решітки N×N |
| `prob_proliferation` (PP) | 0.04 | Ймовірність поділу за крок |
| `prob_migration` (Pm) | 0.02 | Ймовірність міграції за крок |
| `prob_apoptosis` (PA) | 0.005 | Ймовірність апоптозу (лише RTC) |
| `prob_stem_division` (Ps) | 0.01 | Ймовірність симетричного поділу STC |
| `max_proliferation_potential` (pmax) | 10 | Ліміт поділів для RTC |
| `prob_immune_kill` (pdT) | 0.15 | Шанс IC вбити ракову клітину |
| `prob_immune_death` (pdI) | 0.05 | Шанс IC загинути від ракової |
| `immune_migration_speed` | 0.3 | Шанс IC мігрувати за крок |
| `immune_recruit_rate` (ρ) | 0.001 | Коефіцієнт рекрутингу IC |
| `initial_healthy_density` | 0.3 | Початкова частка здорових клітин |
| `initial_immune_count` | 15 | Початкова кількість IC |
| `necrotic_decay_time` | 50 | Кроків до зникнення некрозу |

## 7. Рекрутинг імунних клітин

Нові IC з'являються на краях решітки кожен крок. Кількість нових клітин визначається за формулою Міхаеліса–Ментен:

```
rate = ρ × n_IC(t) × n_cancer(t) / (α + n_cancer(t))
```

де α = 1000, ρ = 0.001. Фактична кількість нових IC за крок:

```
n_new ~ Poisson(max(rate, 0.1))
```

Логіка: чим більше раку → тим активніша імунна відповідь, але з насиченням (при великій кількості раку темп виходить на плато, а не зростає нескінченно).

```
Приклад:
  n_IC=10, n_cancer=50:   rate ≈ 0.000476  → зазвичай 0 нових IC за крок
  n_IC=20, n_cancer=500:  rate ≈ 0.00667   → зрідка 1 нова IC за крок
```

## 8. Алгоритм одного кроку симуляції

```
КРОК t → t+1:

1. Копіюємо поточну решітку (new_grid, new_pot, new_nec)
2. Збираємо всі непорожні клітини → перемішуємо випадково
3. Для кожної клітини у випадковому порядку:
   - NECROTIC    → timer++; якщо timer ≥ 50 → EMPTY
   - HEALTHY     → нічого
   - IMMUNE      → атака або міграція
   - CANCER/STC  → апоптоз / потенціал / поділ / міграція / спокій
4. Рекрутинг нових IC на краях решітки
5. Застосовуємо new_grid (синхронне оновлення)
6. Записуємо статистику
```

Важливо: кожна клітина читає **старий** стан сусідів (`self.grid`), але записує у **новий** (`new_grid`). Це синхронне оновлення — всі зміни застосовуються одночасно після обходу всіх клітин.

## 9. Сценарії симуляції

### `scenario_nonclonogenic` — пухлина від RTC

**Параметри:** одна RTC у центрі, PA=0, без здорових клітин, без IC.

**Очікуваний результат:** пухлина **зникає**. Без STC кожна RTC має обмежений потенціал — рано чи пізно всі клітини вичерпують ліміт поділів і гинуть.

### `scenario_stem` — пухлина від STC (Ps=0)

**Параметри:** одна STC у центрі, PA=0, Ps=0 (без нових STC), без здорових клітин, без IC.

**Очікуваний результат:** **стабільна** пухлина. Одна безсмертна STC безперервно породжує нові RTC, які заповнюють периферію і гинуть. Розмір пухлини виходить на плато.

### `scenario_immune` — повна модель

**Параметри:** одна STC (Ps=2%), здорові клітини (щільність 20%), 20 IC, pdT=20%.

**Очікуваний результат:** IC сповільнюють ріст пухлини, але **не зупиняють** — STC продовжує відтворюватись, а кількість STC з часом зростає через симетричні поділи.

### `scenario_multi` — кілька пухлин

**Параметри:** N пухлин STC рандомно по решітці 120×120, Ps=2%, щільність здорових 20%, 30 IC.

**Особливість:** пухлини ростуть незалежно і з часом можуть злитись. Кількість фактично розміщених пухлин може бути меншою за запитану (якщо не вистачає місця або клітини опиняються надто близько одна до одної — мінімальна дистанція 10 клітин).

## 10. Як запустити

```bash
python3 tumor_ca.py [параметри]
```

Аргументи командного рядка (argparse):

| Аргумент | Значення за замовч. | Опис |
|----------|---------------------|------|
| `--scenario` | `immune` | Сценарій: `nonclonogenic`, `stem`, `immune`, `multi` |
| `--steps` | 300 | Кількість кроків симуляції |
| `--grid` | залежить від сценарію | Розмір решітки |
| `--tumors` | 5 | Кількість пухлин (лише для `multi`) |

Приклади:

```bash
# Стандартний запуск — повна модель з імунною системою
python3 tumor_ca.py

# Неклоногенна пухлина, 200 кроків
python3 tumor_ca.py --scenario nonclonogenic --steps 200

# Стовбурова клітина без імунітету
python3 tumor_ca.py --scenario stem

# Кілька пухлин, велика решітка
python3 tumor_ca.py --scenario multi --tumors 8 --steps 400
```

## 11. Стохастичність

Модель **стохастична** — кожен запуск дає різний результат, бо апоптоз, поділ, міграція та порядок обходу клітин визначаються генератором випадкових чисел. Для надійних висновків потрібно запустити симуляцію кілька разів і усереднити результати.

---

## 12. Контрінтуїтивний ефект: більший апоптоз → більша пухлина

При підвищенні PA (ймовірності апоптозу RTC) пухлина може **зростати швидше**, а не повільніше:

```
PA низький (RTC живуть довго):
  → RTC займають всі місця навколо STC
  → STC не може ділитись (немає вільного місця)
  → пухлина маленька і стабільна

PA високий (RTC швидко гинуть):
  → постійно з'являється вільне місце
  → STC ділиться частіше
  → більше симетричних поділів → більше STC
  → більше STC → пухлина росте агресивніше
```

Це приклад **емерджентної поведінки** — складний макроскопічний ефект з простих мікроскопічних правил.

## 13. Зв'язок з реальною біологією

| Елемент моделі | Реальна біологія |
|----------------|------------------|
| Решітка 100×100 | Тканина ~1 мм² (клітина ≈10 мкм) |
| 1 крок | 1 день |
| RTC, pmax=10 | Ракові клітини з обмеженим числом поділів (ліміт Хейфліка) |
| STC | Ракові стовбурові клітини — джерело живучості пухлини |
| PA = 0.5% | Програмована клітинна смерть (апоптоз) |
| PP ≈ 4% | Клітинний цикл ~24 години |
| IC (IMMUNE) | Т-лімфоцити, NK-клітини та інші ефектори |
| Рекрутинг IC | Імунна відповідь на пухлину |
| NECROTIC | Некротичне ядро пухлини — мертві клітини в центрі |
